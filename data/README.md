# TransferPath Data Pipeline

TransferPath uses a small structured dataset to model universities, transfer requirements, program-level admissions signals, estimated costs, and major-specific competitiveness.

The data pipeline is intentionally separated into raw and processed layers so the project can grow from manually curated data into larger real-world datasets without changing the application code.

## Directory Structure

```text
data/
  raw/
    universities.csv
    programs.csv
  processed/
    universities_clean.csv
    programs_clean.csv
    import_report.txt
```

## Raw Data

The `data/raw` folder stores the human-editable source CSV files.

### `universities.csv`

Each row represents one university.

Important fields:

- `name`
- `country`
- `state`
- `city`
- `min_gpa`
- `application_deadline`
- `website_url`
- `accepts_international`
- `accepts_fall`
- `accepts_spring`
- `application_portal`
- `transfer_requirements`

### `programs.csv`

Each row represents one major/program at a university.

Important fields:

- `university_name`
- `major_name`
- `degree_level`
- `estimated_annual_cost`
- `minimum_recommended_gpa`
- `competitiveness_level`
- `program_url`

## Processed Data

The `data/processed` folder is generated by the Python import pipeline.

Processed files are normalized versions of the raw CSVs. The pipeline checks that:

- required columns exist
- GPA and cost fields are valid numbers
- boolean fields are valid
- URLs use HTTPS
- university names are unique
- every program references a known university
- duplicate university/program rows are rejected

## Running the Pipeline

From the project root:

```bash
python3 scripts/import_universities.py
```

The script generates:

```text
data/processed/universities_clean.csv
data/processed/programs_clean.csv
data/processed/import_report.txt
```

## Application Import

When the Spring Boot app starts, `DataLoader` reads the processed CSV files and imports any missing universities or programs into PostgreSQL.

This means the workflow is:

```text
Edit raw CSVs -> Run Python pipeline -> Start Spring Boot app -> Data loads into PostgreSQL
```

## Data Notes

The current dataset is an MVP-scale curated dataset used to validate the product and ranking system.

It is not yet a comprehensive official admissions dataset. Later phases will expand this pipeline with larger public datasets, clearer source attribution, and richer program-specific signals such as transfer course requirements, deadline windows, and school comparison metrics.
